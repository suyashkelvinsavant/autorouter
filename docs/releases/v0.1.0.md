# AutoRouter v0.1.0

AutoRouter’s first public release provides a local-first gateway that accepts OpenAI, Anthropic, and Gemini wire formats on one endpoint and routes requests to your configured upstream provider.

## Windows installation

Download `AutoRouter_0.1.0_x64-setup.exe`, verify its SHA-256 checksum, then run the installer.

```text
78EDD98E166DD428A02C55498F788CFB1E88E0F0D8CE242B96E88566B8EB3AD9  AutoRouter_0.1.0_x64-setup.exe
```

The installer is not code-signed in this release. Windows may show a SmartScreen warning; verify the checksum before proceeding. Code signing is planned for a future release.

## What’s included

- OpenAI Chat Completions and Responses, Anthropic Messages, and Gemini Generate Content compatibility.
- Streaming, tool use, reasoning-content preservation, and a universal translation pipeline.
- Smart routing, custom providers, model capabilities, and health-based fallback.
- Local desktop dashboard and headless gateway modes.
- Secret-store integration and optional bearer authentication.

## Quick start

1. Start AutoRouter and configure an upstream provider in the dashboard.
2. Point a compatible tool at `http://127.0.0.1:4073`.
3. Set `X-AutoRouter-Source` to `openai`, `anthropic`, or `gemini` where your client requires it.

See the [README](../../README.md) and [manual](../../manual.md) for setup recipes and troubleshooting.

## Notes

- This release includes a freshly built Windows x64 NSIS installer.
- macOS and Linux builds are produced by CI but are not attached to this first release yet.
- MSI packaging is intentionally excluded because WiX cannot reliably write its output from this OneDrive-synced workspace; the NSIS installer is the supported Windows download.
