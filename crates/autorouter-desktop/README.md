# AutoRouter Desktop

`autorouter-desktop` is the Tauri 2 shell that hosts the AutoRouter
gateway and the dashboard UI in a single distributable binary.

## Build

```sh
# Install the Tauri CLI the first time.
npm install --prefix scripts @tauri-apps/cli@^2

# Build platform-native installers.
#   Windows: NSIS + MSI
#   macOS:   .app + .dmg
#   Linux:   .deb + .AppImage
node scripts/bundle.mjs
```

The output lands under `target/release/bundle/`.

`node scripts/rebuild.js` is a convenience wrapper for local
iteration: it installs the UI deps (if missing), rebuilds the UI,
then runs the Tauri bundler in release mode.

## Run from source

```sh
# Build the web UI once.
cd ui && npm install && npm run build && cd ..

# Run the desktop binary in dev mode (it spawns the gateway in-process).
cargo run -p autorouter-desktop
```

## UI

The web UI is a Vite + React + TypeScript single-page app under
`ui/`. It communicates with the gateway over the in-process JSON
API:

* `GET  /ui/status`
* `GET  /ui/providers`
* `GET  /ui/sessions`
* `GET  /ui/settings` / `PATCH /ui/settings`
* `GET  /ui/logs?since=...&level=...&limit=...`
* `POST /ui/restart`
* `GET  /ui/server`

When running inside the Tauri shell the UI also calls the Rust
commands directly via `invoke()` (see
`crates/autorouter-desktop/src/lib.rs` and
`ui/src/api.ts`).

## Keyboard shortcuts

| Shortcut         | Action              |
| ---------------- | ------------------- |
| `Ctrl/Cmd + 1`   | Dashboard           |
| `Ctrl/Cmd + 2`   | Providers           |
| `Ctrl/Cmd + 3`   | Sessions            |
| `Ctrl/Cmd + 4`   | Logs                |
| `Ctrl/Cmd + 5`   | Settings            |
| `Ctrl/Cmd + ,`   | Settings            |
| `Ctrl/Cmd + R`   | Refresh status      |

The active page is persisted in `localStorage` and reflected in
the URL as `?page=dashboard|providers|sessions|logs|settings`,
so any page can be deep-linked from an external tool.

## Installer

The bundler is configured in
`crates/autorouter-desktop/tauri.conf.json`. Targets are
auto-selected by `scripts/bundle.mjs` based on the host platform.

### Windows

* **NSIS** (`AutoRouter_<version>_x64-setup.exe`) - per-machine
  setup binary. Recommended for most users.
* **MSI** (`AutoRouter_<version>_x64_en-US.msi`) - generated via
  WiX. Use this for Group Policy deployment or `msiexec`.

The Windows bundle is built with the **WebView2 bootstrapper
embed** install mode, so end users do not need a separate
WebView2 install.

### macOS

* **`.app`** - the application bundle (`AutoRouter.app`).
* **`.dmg`** - drag-to-Applications disk image.

Two files control the Mac bundle:

* `info/Info.plist` - the bundle metadata (display name, copyright,
  minimum macOS version 10.15, ATS exception for the local
  gateway, the developer-tools LSApplicationCategoryType).
* `info/entitlements.plist` - the hardened-runtime entitlements
  (network client + server, user-selected file read, JIT, and
  library validation disabled). These are merged into the signed
  binary when signing is enabled.

The hardened runtime is on by default. For internal / unsigned
builds you can pass `AUTOROUTER_SKIP_SIGNING=1`; for production
releases set `APPLE_SIGNING_IDENTITY`, `APPLE_ID`,
`APPLE_PASSWORD`, and `APPLE_TEAM_ID` to the values from your
Apple Developer account.

### Linux

* **`.deb`** - Debian / Ubuntu package.
* **`.AppImage`** - portable single-file binary.

## Code signing

The build is **unsigned** by default. To enable signing:

* **Windows** - set `TAURI_SIGNING_PRIVATE_KEY` and
  `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`, or use the
  `bundle.windows.certificateThumbprint` field with the certificate
  in the CurrentUser store. Tauri's signing guide is at
  <https://tauri.app/distribute/sign/windows/>.
* **macOS** - configure `bundle.macOS.signingIdentity` to your
  Apple Developer ID, and provide `APPLE_ID`,
  `APPLE_PASSWORD` (app-specific), and `APPLE_TEAM_ID` env vars.
  Use `AUTOROUTER_SKIP_SIGNING=1` to opt out for internal builds.
  After signing, run `xcrun notarytool submit` to notarize.
* **Linux** - signing is opt-in via `gpg` and is not required for
  most distributions.

## Tray icon

The system tray icon is embedded at build time from
`icons/32x32.png` (generated by `scripts/gen-png.js`). After
editing the brand mark, re-run:

```sh
node scripts/gen-png.js
node scripts/gen-ico-icns.js
```

then rebuild the bundle.

## Tray menu

* **Open Dashboard** - shows and focuses the main window.
* **Open Logs** - raises the window and switches to the Logs page.
* **Quit AutoRouter** - shuts the gateway and exits.

Closing the main window hides it to the tray; the gateway keeps
running until you explicitly quit.
